# OpenClaw Skill Rule Audit Ledger

## 1. Purpose

This ledger is the skills-side audit baseline for heuristic and rule-heavy skills. It does not mark any rule as approved. Batch 1 uses it to freeze default-chain usage and make every listed skill explicitly approval-gated before execution.

## 2. Ledger Schema

Each audit record must eventually contain:

1. `rule_id`
2. `skill`
3. `rule_text`
4. `trigger_conditions`
5. `exception_conditions`
6. `known_biases`
7. `representativeness_risks`
8. `sample_input`
9. `sample_output`
10. `audit_status`
11. `effective_version`

## 3. Batch 1 Initial Records

Status values used here:

1. `default-frozen`: not eligible for default investigator, moderator, source-queue, or controller chains.
2. `approval-required`: execution requires `skill_approval_requests -> approval -> run-skill --skill-approval-request-id`.
3. `audit-pending`: detailed rule review is still pending; the record is a freeze-line placeholder, not an approval.
4. `legacy-isolated`: current semantics are considered legacy or deprecated until rewritten.

## 4. Batch 1 Freeze Line

1. No listed skill may appear in default agent entry recommendations or source-queue downstream hints.
2. No listed skill may execute through `run-skill` or `preflight-skill` without an approved, unconsumed skill approval request.
3. Outputs from listed skills are derived advisory or legacy helper surfaces until a later audit record explicitly changes their status.
